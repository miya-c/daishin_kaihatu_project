# 月次処理 误操作防止・修正機能 設計書

## Phase 1: 予防（フロントエンド）

### 1-1. デフォルト年月（変更可）
- 初期値は現在の年月
- 変更自由、選択中の年月を常に大きく表示

### 1-2. 確認ラベル強化
- 「2026年5月分として月次処理を実行します」を目立つ場所に常時表示
- 年月変更でリアルタイム更新

### 1-3. 二段階確認（既存動作維持）
- 事前チェック → 確認 → 実行

## Phase 2: バックアップ方式変更（バックエンド）

### 2-1. _monthly_backup 固定シートへ変更
- `inspection_data_pre_reset_*` 新規作成 → `_monthly_backup` 上書き

### 2-2. 物件マスタ完了日もバックアップ
- `_monthly_backup` に inspection_data + 物件マスタ完了日列を保存

### 2-3. メタデータ記録
- 実行年月・実行日時を実行行として保存

### 2-4. 古いバックアップシートクリーンアップ
- 月次処理時に `inspection_data_pre_reset_*` を削除

### _monthly_backup シート構成:
```
行1: [META] 実行年月=2026-05, 実行日時=2026-04-22T15:30:00
行2以降: [INSPECTION_DATA] inspection_data の全データ（ヘッダー付き）
空行
物件マスタヘッダー行
物件マスタデータ（物件ID + 検針完了日のみ）
```

## Phase 3: 修正機能（バックエンド + フロントエンド）

### バックエンド

#### 3-1. getMonthlyBackupInfo API
- _monthly_backup の有無・メタデータを返す

#### 3-2. undoMonthlyProcess API
- LockService取得
- _monthly_backup メタデータ読み込み
- ユーザー確認（フロント側で実施）
- アーカイブシートから該当月データ削除（空になったらシート削除）
- inspection_data をバックアップから復元
- 物件マスタ完了日をバックアップから復元
- LockService解放

### フロントエンド

#### 3-3. 取り消しボタン追加
- getMonthlyBackupInfo で動的表示
- バックアップがない場合は非表示

#### 3-4. 確認ダイアログ
- 「〇年△月分の月次処理を取り消します」+ 二段階確認
- 制限事項明記（直近1回分のみ）

#### 3-5. 結果表示
- 完了後、正しい月で再実行を促す

## デプロイ
- Phase 2 + 3 は同時デプロイ

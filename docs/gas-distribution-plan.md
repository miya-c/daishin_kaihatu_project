# GAS版 配布計画書

## 1. 配布アーキテクチャ

### 2層モデル: ライブラリ + クライアント

```
┌─────────────────────────────────────────────────────────────┐
│  ライブラリプロジェクト（開発者管理）                           │
│  backend/水道検針ライブラリ/                                   │
│  ├── api_data_functions.js                                    │
│  ├── property_management.js                                   │
│  ├── data_management.js                                       │
│  ├── web_app_admin_api.js                                     │
│  ├── web_app_api.js                                           │
│  └── ...（全19ファイル）                                       │
│                                                               │
│  → GAS ライブラリとして公開（スクリプトID: XXXXXXX）           │
│  → 識別子: cmlibrary                                          │
└─────────────────────────────────────────────────────────────┘
                          ↑ ライブラリ参照
┌─────────────────────────────────────────────────────────────┐
│  クライアントプロジェクト（各社のGoogleアカウント）              │
│                                                               │
│  ┌─── Google Spreadsheet ──────────────────────────────┐    │
│  │  物件マスタ | 部屋マスタ | inspection_data | ...      │    │
│  └──────────────────────────────────────────────────────┘    │
│                          ↑ バインド                           │
│  ┌─── GAS プロジェクト ─────────────────────────────────┐    │
│  │  library_client.gs  ← 1ファイルだけ貼り付け            │    │
│  │  ライブラリ追加: cmlibrary（スクリプトID指定）           │    │
│  └──────────────────────────────────────────────────────┘    │
│                          ↑ Web App デプロイ                    │
│  ┌─── 管理画面 ─────────────────────────────────────────┐    │
│  │  https://script.google.com/macros/s/XXXXX/exec        │    │
│  │  admin.html を serve（doGet が HTML を返す）            │    │
│  └──────────────────────────────────────────────────────┘    │
│                                                               │
│  ┌─── PWA（Cloudflare Pages）───────────────────────────┐    │
│  │  VITE_GAS_WEB_APP_URL = 上記 Web App URL               │    │
│  │  VITE_GAS_API_KEY = スクリプトプロパティの API_KEY      │    │
│  └──────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

### 各社の独立性

| 項目 | 各社が所有 |
|---|---|
| Googleアカウント | ✅ 各社 |
| Google Spreadsheet | ✅ 各社のGoogle Drive |
| GASプロジェクト | ✅ 各社 |
| データ | ✅ 各社 |
| ライブラリ | ❌ 開発者が管理（スクリプトIDで参照） |
| PWA（Cloudflare） | ✅ 各社（または共通） |

## 3. 新規導入手順（チェックリスト）

### Phase A: 各社のGoogleアカウント準備

- [ ] **A1**: 各社のGoogleアカウントでGoogle Spreadsheetを作成
- [ ] **A2**: スプレッドシート名を「水道検針データ」等に設定

### Phase B: GASプロジェクト設定

- [ ] **B1**: スプレッドシートから「拡張機能 → Apps Script」を開く
- [ ] **B2**: デフォルトの「コード.gs」の内容を全て削除
- [ ] **B3**: `library_client.gs` の内容をコピー＆ペースト
- [ ] **B4**: 「ライブラリ」→「ライブラリを追加」→ スクリプトIDを入力 → 識別子を `cmlibrary` に設定
- [ ] **B5**: プロジェクト設定（歯車アイコン）→ スクリプトプロパティに以下を追加:
  - `API_KEY` = `sk-` + ランダム文字列（例: `openssl rand -hex 20` で生成）
  - `ADMIN_TOKEN` = `admin-` + ランダム文字列

### Phase C: Web App デプロイ

- [ ] **C1**: 「デプロイ」→「新しいデプロイ」
- [ ] **C2**: 種類: ウェブアプリ
- [ ] **C3**: 実行するバージョン: 新規
- [ ] **C4**: 実行 as: 自分
- [ ] **C5**: アクセスできるユーザー: 全員
- [ ] **C6**: デプロイ → URLをコピー

### Phase D: 初期セットアップ（スプレッドシート）

- [ ] **D1**: Apps Scriptエディタで「セットアップウィザード」を実行
  - メニュー → 「🔧 システム設定」→「セットアップウィザード」
  - または管理画面の「初期設定」タブから実行
- [ ] **D2**: 以下が自動作成されることを確認:
  - 物件マスタシート
  - 部屋マスタシート
  - inspection_dataシート
  - 設定シート（config）

### Phase E: 動作確認

- [ ] **E1**: ブラウザで `{GAS_URL}?action=test` にアクセス
  - → `{"success":true,"message":"ライブラリAPI接続テスト成功"}` が返ること
- [ ] **E2**: Apps Scriptエディタで `testLibraryConnection()` を実行
  - → ライブラリ接続が正常であること
- [ ] **E3**: 管理画面を開く（`{GAS_URL}` にブラウザでアクセス）
  - → ログイン画面が表示されること
  - → ADMIN_TOKENでログインできること
  - → ダッシュボードが表示されること
- [ ] **E4**: 管理画面で物件を1件追加
  - → スプレッドシートの物件マスタに反映されること
- [ ] **E5**: 管理画面で部屋を追加
  - → スプレッドシートの部屋マスタ・inspection_dataに反映されること

### Phase F: PWA設定（オプション・各社）

- [ ] **F1**: PWAの環境変数を設定:
  - `VITE_GAS_WEB_APP_URL` = C6でコピーしたURL
  - `VITE_GAS_API_KEY` = B5で設定したAPI_KEY
- [ ] **F2**: PWAをビルド・デプロイ（Cloudflare Pages等）
- [ ] **F3**: PWAで物件一覧が表示されることを確認
- [ ] **F4**: PWAで検針データの保存ができることを確認

## 4. ライブラリ更新時の配布手順

### 開発者がライブラリを更新した場合

各社のクライアントプロジェクトは**ライブラリを参照しているだけ**なので、ライブラリ側を更新するだけで全社に反映される。

```
開発者: ライブラリのコードを変更 → GASエディタで「保存」
  → 全社のクライアントプロジェクトが次回実行時に自動的に最新版を使用
```

**注意**: ライブラリの更新は即座に反映されるが、既に実行中のインスタンスには古いバージョンがキャッシュされる場合がある。数分以内に反映される。

### admin.html（管理画面）を更新した場合

```bash
cd admin-ui && npm run build && cp dist/index.html ../backend/水道検針ライブラリ/admin.html
```

ライブラリプロジェクトでadmin.htmlを更新 → 保存 → 各社のWeb App URLにアクセスすると最新版が表示される。

## 5. トラブルシューティング

### よくある配布時の問題

| 問題 | 原因 | 対処 |
|---|---|---|
| ライブラリ関数が undefined | 識別子が `cmlibrary` になっていない | ライブラリ設定で識別子を確認 |
| API_KEY エラー | スクリプトプロパティ未設定 | B5の手順を再確認 |
| 管理画面が表示されない | admin.htmlがライブラリに含まれていない | ライブラリの最新版を使用 |
| シートが存在しないエラー | 初期セットアップ未実行 | D1の手順を実行 |
| Web Appが404 | デプロイが古い | C1〜C6を再実行（新しいデプロイ） |

### デバッグ手順

1. Apps Scriptエディタで `testLibraryConnection()` を実行
2. ログでエラー内容を確認
3. `showAvailableFunctions()` でライブラリ関数が認識されているか確認

## 7. スプレッドシート初期化仕様

### 必須シート（4つ）

| シート名 | 用途 | 定義元 |
|---|---|---|
| `物件マスタ` | 物件情報 | `CONFIG.SHEET_NAMES.PROPERTY_MASTER` |
| `部屋マスタ` | 部屋情報 | `CONFIG.SHEET_NAMES.ROOM_MASTER` |
| `inspection_data` | 検針データ（現在月） | `CONFIG.SHEET_NAMES.INSPECTION_DATA` |
| `設定値` | システム設定 | `CONFIG.SHEET_NAMES.SETTINGS` |

### ヘッダー定義（正確な列名）

**物件マスタ**（3列）

| 列 | ヘッダー | 型 |
|---|---|---|
| A | 物件ID | 文字列（P000001形式） |
| B | 物件名 | 文字列 |
| C | 検針完了日 | 日付 |

**部屋マスタ**（5列）

| 列 | ヘッダー | 型 |
|---|---|---|
| A | 物件ID | 文字列（物件マスタ参照） |
| B | 部屋ID | 文字列（P000001-R001形式） |
| C | 部屋名 | 文字列 |
| D | 部屋ステータス | 文字列（normal/skip等） |
| E | 備考 | 文字列 |

**inspection_data**（15列）

| 列 | ヘッダー | 型 |
|---|---|---|
| A | 記録ID | 文字列 |
| B | 物件名 | 文字列 |
| C | 物件ID | 文字列 |
| D | 部屋ID | 文字列 |
| E | 部屋名 | 文字列 |
| F | 検針日時 | 日時 |
| G | 警告フラグ | 文字列 |
| H | 標準偏差値 | 数値 |
| I | 今回使用量 | 数値 |
| J | 今回の指示数 | 数値 |
| K | 前回指示数 | 数値 |
| L | 前々回指示数 | 数値 |
| M | 前々々回指示数 | 数値 |
| N | 検針不要 | 真偽値 |
| O | 請求不要 | 真偽値 |

> 実行時に `部屋ステータス` 列が追加される場合あり（16列目）

**設定値**（3列）

| 列 | ヘッダー | 型 |
|---|---|---|
| A | 設定項目 | 文字列 |
| B | 設定値 | 文字列 |
| C | 説明 | 文字列 |

### 設定値デフォルト

| 設定項目 | 設定値 | 説明 |

## 8. 導入ウィザード（7ステップ）

セットアップウィザード（`水道検針アプリ.js` の `defineSetupSteps()`）:

| ステップ | 名称 | 実行関数 | 内容 |
|---|---|---|---|
| 1 | システム診断・環境確認 | `executeSystemDiagnosis` | スプレッドシート・権限の確認 |
| 2 | マスタシートテンプレート作成 | `executeTemplateCreation` | 4シートのヘッダー作成 |
| 3 | サンプルデータ投入 | `executeSampleDataCreation` | サンプル物件・部屋データの投入（スキップ可） |
| 4 | 物件ID自動割り当て | `executePropertyIdAssignment` | 物件IDのフォーマット（P000001形式） |
| 5 | 部屋ID自動生成 | `executeRoomIdGeneration` | 部屋IDの自動生成 |
| 6 | 検針データシート作成 | `executeInspectionDataCreation` | inspection_dataの初期行生成 |
| 7 | 最終確認・動作テスト | `executeFinalValidation` | 全体の整合性チェック |

起動方法: GASメニュー → 「🔧 システム設定」→「導入ウィザード」

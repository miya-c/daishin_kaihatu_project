# 水道検針アプリ パフォーマンス改善 TODO

## 現在の状況分析
- フロントエンド: PWA対応のReactアプリケーション（CDN経由）
- バックエンド: GAS + Google Spreadsheet
- 主要課題: レンダリング最適化、API応答時間、リソース読み込み

## 優先度順改善項目

### 🔴 最優先（即座に実施）

#### 1. レイジーロード実装
- **現状:** 物件一覧を一度に全て読み込み
- **改善案:** 
  - 仮想スクロール実装（10-20件ずつ表示）
  - Intersection Observer API活用
  - 検索フィルタリング前のデータ最小化
- **実装場所:** `property_select.html`の`filteredProperties`処理

#### 2. コンポーネント最適化
- **現状:** 物件カードが全て同時レンダリング
- **改善案:**
  - React.memo()でメモ化
  - useCallback()でイベントハンドラ最適化
  - 不要なre-renderの削減
- **実装場所:** property_select.html内のReactコンポーネント

#### 3. 不要なレンダリング削減
- **現状:** 検索時に全物件を再評価
- **改善案:**
  - useMemo()で検索結果をメモ化
  - debounce機能で検索頻度制限
  - 状態管理の最適化
- **実装場所:** `handleSearchChange`関数

### 🟡 重要（短期実施）

#### 4. イベントハンドラ最適化
- **現状:** 300ms遅延 + 複数の状態更新
- **改善案:**
  - `setTimeout(300ms)`の削除
  - 状態更新の統合
  - 非同期処理の最適化
- **実装場所:** `handlePropertySelect`関数

#### 5. APIレスポンス時間短縮
- **現状:** 物件データ + 部屋データを順次取得
- **改善案:**
  - 必要最小限のデータのみ取得
  - 部屋データの遅延読み込み
  - GAS Cache Service活用
- **実装場所:** バックエンドGASスクリプト

#### 6. キャッシュ利用強化
- **現状:** `cache=${Date.now()}`でキャッシュ無効化
- **改善案:**
  - SessionStorage/LocalStorage活用
  - HTTP キャッシュヘッダー適用
  - サービスワーカーでのキャッシュ戦略
- **実装場所:** 全体的なデータフロー

### 🟢 改善（中期実施）
